> 規劃日期：2026-06-11。
> 進度（2026-06-11）：第一階段完成＋第二階段的出題設定頁與選項亂序也已完成。
> 已完成：三格式匯入與驗證、IndexedDB 儲存、出題設定（題型/選項數/標籤/題數/隨機/選項亂序）、答題流程（即時判定＋解析）、結算頁、Excel 範本下載、作答統計已寫入 stats 表（供錯題本用）。
> 待做：錯題本頁面與錯題複習模式（第三階段）、JSON 匯出備份、加分項。

## 需求摘要

- 個人刷題用，只有自己使用
- 自行匯入固定格式題庫：**Excel / CSV / JSON** 三種格式
- 可選擇出題條件：單選/多選、選項數、標籤/章節
- 功能：錯題本與複習、答題解析顯示、選項隨機排序

## 一、技術架構

- **框架**：Vue 3（Composition API）+ Vite，Pinia 管狀態
- **UI**：Naive UI 或 Element Plus（或 Tailwind 自刻，待定）
- **Excel/CSV 解析**：SheetJS (xlsx)，純瀏覽器端解析
- **資料保存**：IndexedDB（用 Dexie.js 包裝）存題庫和錯題紀錄
- **部署**：純靜態檔，可直接開 index.html，或放 PC 上經 Tailscale 從手機/iPad 刷題

## 二、題庫格式規格

### Excel / CSV（一行一題）

| 欄位 | 必填 | 說明 |
|------|------|------|
| 題目 | ✅ | 題幹文字 |
| A ~ F | A、B 必填 | 選項內容，留空視為無此選項（每題 2~6 個選項） |
| 答案 | ✅ | 如 `B` 或 `AC`；一個字母＝單選，多個＝多選 |
| 解析 | 選填 | 答題後顯示 |
| 標籤 | 選填 | 多個用逗號分隔，如 `內科,心臟` |
| 選項固定 | 選填 | 標記該題選項不隨機打亂 |

### JSON

```json
{
  "name": "內科題庫",
  "questions": [
    {
      "question": "下列何者正確？",
      "options": ["甲", "乙", "丙", "丁"],
      "answer": [0, 2],
      "explanation": "因為…",
      "tags": ["內科", "心臟"],
      "fixedOrder": false
    }
  ]
}
```

- `answer` 用選項索引陣列（0 起算）：一個元素＝單選，多個＝多選
- 相容 `"answer": "AC"` 字母寫法，匯入時自動轉換
- `explanation`、`tags`、`fixedOrder` 選填
- **題庫匯出備份**用同一個 schema（備份/還原、跨裝置搬移）

### 匯入驗證（三種格式共用同一套流程）

- 答案對應的選項不能是空的／索引不能超出範圍
- 缺題目、缺答案 → 列出問題清單（標出第幾行/第幾題），不默默跳過
- 匯入成功顯示統計：共幾題、單選/多選各幾題、各標籤題數
- 多次匯入：可「覆蓋整個題庫」或「新增為另一個題庫」，多份題庫並存

## 三、出題設定

1. 選題庫（多份時）
2. 題型篩選：全部 / 只出單選 / 只出多選
3. 選項數篩選：依每題實際選項數過濾
4. 標籤篩選：勾選章節範圍
5. 題數與順序：題目數量（或全部）、題目隨機排序、**選項隨機排序（預設開）**
6. 出題來源：全部題目 / 只出錯題本 / 優先出做錯次數多的題

設定時即時顯示「符合條件的題目共 N 題」。

### 選項亂序細節

- 打亂發生在每次出題時，不動原始題庫；答案判定用內部索引，與顯示順序無關
- 匯入時自動偵測「以上皆是/以上皆非」類選項 → 該題標記不打亂（或固定該選項在最後）
- 可用 `fixedOrder` / 「選項固定」欄手動指定
- 錯題本記錄存選項**內容**而非位置

## 四、答題與複習

- 單選點了立即判定；多選按「確認」後判定
- 答錯立即顯示正確答案 + 解析；進度條顯示第幾題/共幾題、答對率
- 多選題計分：全對才算對（不做部分給分）
- 錯題本：記錄做過幾次、錯幾次、最後作答時間與選項；連續答對 N 次（如 2 次）自動移除；可瀏覽、手動移除、看常錯標籤
- 結算頁：本輪答對率、錯題清單（可展開看解析），錯題自動入錯題本

## 五、頁面結構

```
首頁（題庫總覽、統計）
├── 匯入頁：拖拉/選擇 Excel/CSV/JSON・驗證結果・匯入預覽
├── 出題設定頁：題型/選項數/標籤/題數/順序
├── 答題頁：一題一畫面、即時解析
├── 結算頁：成績、錯題回顧
└── 錯題本頁：瀏覽、管理、啟動錯題複習
```

## 六、實作順序

1. **第一階段（可用雛形）**：專案骨架 → Excel/CSV/JSON 解析與共用驗證 → IndexedDB 存題庫 → 基本答題流程（單選/多選、立即判定、顯示解析）
2. **第二階段**：出題設定頁（題型/選項數/標籤/隨機/選項亂序）→ 結算頁 → JSON 匯出備份
3. **第三階段**：錯題本、錯題複習、答題統計
4. **加分項**：間隔重複（類 Anki 排程）、考試模式（計時、交卷後才看答案）

## 已採取的預設（可再改）

- 選項上限 6 個（A~F）
- 多選全對才給分
- 資料存瀏覽器 IndexedDB，換瀏覽器/清快取會不見 → 靠 JSON 匯出備份
